# ETA Project Constitution

## Purpose

This Constitution defines the immutable principles governing the ETA Ecosystem.

Every business, architectural, technical, operational and AI decision must comply with this document.

This Constitution overrides temporary implementation decisions, conversations, coding preferences and architectural opinions.

It is the highest governance document of the ETA Project.

---

# Mission of ETA

ETA exists to become the Intelligent Operating System of Industrial Procurement.

Instead of being only a steel trading company, ETA is evolving into a technology-enabled procurement ecosystem connecting customers, suppliers, manufacturers, enterprise systems and Artificial Intelligence through one integrated platform.

---

# Vision

Build the most intelligent industrial procurement ecosystem in the Middle East.

An ecosystem where procurement becomes:

- Faster
- Smarter
- Transparent
- AI Driven
- Data Driven
- Knowledge Driven
- Enterprise Connected

---

# Core Philosophy

We are not building:

- a website
- a CRM
- an ERP

We are building an Enterprise Procurement Ecosystem.

Every module must strengthen every other module.

The whole platform must always be greater than the sum of its parts.

---

# Documentation Strategy

Documentation is never optional.

Every feature implemented inside ETA Platform must first exist inside ETA Blueprint.

ETA Blueprint is the Documentation Source of Truth.

ETA Platform is the Implementation Source of Truth.

If it is not documented, it does not exist.

---

# Knowledge Strategy

All business knowledge belongs inside ETA Blueprint.

Obsidian acts as the Enterprise Brain.

GitHub acts as the Permanent Repository.

No business knowledge should exist only inside AI conversations.

Knowledge must always become documentation.

---

# Git Strategy

GitHub is the single source of truth.

Repositories include:

- ETA-Blueprint
- ETA-Platform

Rules:

- Never recreate repositories.
- Never overwrite completed work.
- Never duplicate documentation.
- Always evolve existing architecture.
- Every major milestone must be committed and versioned.

---

# AI Strategy

- OpenAI

AI recommends.
