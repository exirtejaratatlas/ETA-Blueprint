# RFQ Lifecycle

## Purpose

The RFQ Lifecycle defines the complete procurement journey from the creation of an RFQ until award, purchase order creation, and archival.

It ensures a controlled, auditable, and AI-assisted sourcing process.

# Stage 1 — Idea

The sourcing requirement originates from:

- Customer Request
- Opportunity
- Project
- Maintenance
- Inventory Replenishment
- AI Recommendation

Status

```
Idea
```

---

# Stage 2 — Draft

Procurement creates the initial RFQ.

Required:

- Customer
- Product Lines
- Quantities
- Scope

Status

```
Draft
```

---

# Stage 3 — Engineering Review

Engineering validates:

- Technical Specifications
- Drawings
- Datasheets
- Standards
- Material Requirements

Status

```
Engineering Review
```

---

# Stage 4 — Procurement Review

Procurement validates:

- Supplier List
- Manufacturer
- Procurement Strategy
- Budget
- Lead Time

Status

```
Procurement Review
```

---

# Stage 5 — Compliance Review

Compliance validates:

- Export Control
- Sanctions
- Certifications
- Country Restrictions

Status

```
Compliance Review
```

---

# Stage 6 — Approved

The RFQ is formally approved.

Required approvals:

- Engineering
- Procurement
- Compliance

High-value RFQs may also require Executive approval.

Status

```
Approved
```

---

# Stage 7 — Supplier Invitation

Approved Suppliers receive invitations.

Supported channels:

- Email
- Supplier Portal
- API
- EDI

Status

```
Sent
```

---

# Stage 8 — Supplier Responses

Suppliers submit:

- Commercial Proposal
- Technical Proposal
- Attachments
- Delivery Schedule

Status

```
Supplier Responding
```

---

# Stage 9 — Technical Evaluation

Engineering evaluates:

- Compliance
- Technical Match
- Deviations
- Alternatives

Status

```
Technical Evaluation
```

---

# Stage 10 — Commercial Evaluation

Procurement compares:

- Price
- Delivery
- Payment Terms
- Warranty
- Incoterms
- Supplier Performance

Status

```
Commercial Evaluation
```

---

# Stage 11 — Award Decision

Selected supplier(s) are approved.

Possible outcomes:

- Single Award
- Split Award
- Rejected
- Cancelled

Status

```
Awarded
```

---

# Stage 12 — Purchase Order Creation

Awarded quotations generate:

- Purchase Orders
- Contract References
- Supplier Commitments

Status

```
PO Created
```

---

# Stage 13 — Closed

RFQ is completed.

No further quotations are accepted.

Status

```
Closed
```

---

# Stage 14 — Archived

Historical RFQs remain available for:

- Audit
- Analytics
- AI Training
- Procurement Benchmarking

Status

```
Archived
```

# Long-Term Vision

The RFQ Lifecycle provides a standardized enterprise sourcing workflow that integrates engineering, procurement, compliance, suppliers, ERP, and AI into a single governed procurement process.
