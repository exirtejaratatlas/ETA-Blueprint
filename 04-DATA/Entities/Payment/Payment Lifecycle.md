# Payment Lifecycle

## Purpose

The Payment Lifecycle defines the complete journey of a payment from request through approval, treasury execution, banking confirmation, accounting reconciliation, and archival.

The lifecycle guarantees financial governance, treasury control, banking compliance, accounting integrity, AI-assisted cash flow management, and complete enterprise auditability.

---

Requested

---

# Stage 1 — Draft

The payment is created.

Activities

- Create payment request
- Select supplier/customer
- Reference invoice
- Calculate payment amount

Status

```
Draft
```

---

# Stage 2 — Requested

Payment request is submitted.

Activities

- Submit request
- Lock payment details
- Notify treasury

Status

```
Requested
```

---

# Stage 3 — Treasury Review

Treasury validates liquidity and banking information.

Activities

- Cash availability check
- Bank account verification
- Currency validation
- Treasury recommendation

Status

```
Treasury Review
```

---

# Stage 4 — Financial Approval

Payment is reviewed.

Activities

- Finance approval
- Management approval
- Executive approval (when required)

Status

```
Pending Approval
```

---

# Stage 5 — Approved

Payment is authorized.

Activities

- Approval completed
- Payment released
- Treasury authorization

Status

```
Approved
```

---

# Stage 6 — Payment Processing

Execution begins.

Activities

- Bank transfer
- TT
- LC
- Cheque
- Cash settlement
- Internal transfer

Status

```
Processing
```

---

# Stage 7 — Bank Confirmation

Financial institution confirms execution.

Activities

- Receive bank confirmation
- SWIFT verification
- Payment receipt registration

Status

```
Bank Confirmed
```

---

# Stage 8 — Completed

Payment has successfully settled.

Activities

- Update invoice balance
- Update treasury
- Notify accounting

Status

```
Completed
```

---

# Stage 9 — Reconciled

Accounting reconciliation occurs.

Activities

- Journal verification
- Bank reconciliation
- Ledger reconciliation

Status

```
Reconciled
```

---

# Stage 10 — Closed

Payment process is finalized.

Activities

- Close financial transaction
- KPI calculation
- Financial reporting

Status

```
Closed
```

---

# Stage 11 — Archived

Historical payment remains available.

Activities

- Financial audit
- Treasury analysis
- AI learning
- Executive reporting
- Compliance review

Status

```
Archived
```
