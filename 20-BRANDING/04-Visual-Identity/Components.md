# ETA Component Design Language

## Purpose

This document defines the visual behavior of reusable UI components across the ETA ecosystem.

All interfaces—including Website, Dashboard, Supplier Portal, Manufacturer Portal, AI Workspace, and Mobile Applications—must follow these standards.

---

# Design Principles

Every component must be:

- Simple
- Consistent
- Accessible
- Responsive
- Reusable
- Enterprise Grade

---

# Buttons

Types

- Primary
- Secondary
- Outline
- Ghost
- Danger
- Success

States

- Default
- Hover
- Focus
- Active
- Disabled
- Loading

Primary buttons use ETA Navy.

Important actions may use ETA Copper.

---

# Cards

Cards are the primary information container.

Characteristics:

- Large spacing
- Rounded corners (12–16px)
- Soft shadow
- Clear hierarchy
- Optional icon
- Optional KPI

---

# Forms

Form controls include:

- Text Input
- Search
- Select
- Multi Select
- Date Picker
- Upload
- Checkbox
- Radio
- Toggle
- Text Area

Validation must always provide clear visual feedback.

---

# Tables

Enterprise tables support:

- Sorting
- Filtering
- Pagination
- Sticky Header
- Row Selection
- Bulk Actions
- Export

---

# Navigation

Navigation components include:

- Top Navigation
- Sidebar
- Breadcrumb
- Tabs
- Context Menu
- Command Palette

---

# Status Components

Status should always be represented by both:

- Color
- Icon
- Text Label

Statuses include:

- Success
- Warning
- Error
- Info
- Pending

---

# Feedback Components

- Toast
- Alert
- Modal
- Drawer
- Confirmation Dialog
- Empty State
- Skeleton Loader
- Progress Indicator

---

# KPI Components

Enterprise KPI cards display:

- Value
- Label
- Trend
- Comparison
- Icon
- Time Period

---

# AI Components

Specialized components include:

- AI Chat
- AI Suggestions
- AI Actions
- AI Recommendations
- Knowledge Results
- Prompt Input
- Workflow Timeline

These components define the visual identity of ETA's AI-native experience.

---

# Responsive Behavior

Every component must adapt gracefully to:

- Mobile
- Tablet
- Laptop
- Desktop
- Ultra-wide Displays

---

# Design Goal

Every ETA interface should feel:

- Premium
- Fast
- Calm
- Intelligent
- Reliable
- Engineering Driven

The user experience should communicate confidence before the first interaction.
